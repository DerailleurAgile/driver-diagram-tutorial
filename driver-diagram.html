Driver Diagram

Improvement Science — Learning Aid

A Driver Diagram is a structured map of your theory of improvement — before you act on it. It shows what you believe needs to change in a system, and why, in a form that a whole team can see, challenge, and test.

It is not a plan. It is a set of hypotheses. Every arrow is a claim: if we change this, we expect that to improve. The diagram makes those claims explicit so they can be examined — and so that PDSA cycles can test them.

Deming connection: Deming's Theory of Knowledge holds that action without theory is blind — you cannot learn from results unless you first predicted them. A Driver Diagram operationalises this: it externalises the team's working theory so that data can speak to it. It is System of Profound Knowledge made visible.

The worked example

We will build a Driver Diagram together, using a real example from Learning to Improve (Bryk et al., 2015).
